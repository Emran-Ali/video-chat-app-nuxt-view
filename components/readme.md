# Components Folder Structure

The `components` folder is organized to maintain a clean and scalable architecture for the project. Below is the structure and a brief description of each subfolder:

```
components/
├── base/
├── shared/
├── views/
│   ├── auth/
│   ├── dashboard/
│   └── user-profile/
└── forms/
```

## Folder Descriptions

### base/

Contains fundamental components that are used throughout the application, such as buttons, inputs, and other basic UI elements.

### layout/

Includes components related to the layout of the application, such as headers, footers, sidebars, and grid systems.

### shared/

Houses components that are shared across multiple views or features, such as modals, notifications, and loaders.

### views/

This folder is divided into subfolders representing different sections or pages of the application.

- **auth/**: Components related to authentication, such as login, registration, and password recovery.
- **dashboard/**: Components specific to the dashboard view, including widgets, charts, and statistics.
- **user-profile/**: Components for the user profile section, such as profile details, settings, and activity logs.

### forms/

Contains form-related components, including form controls, validation messages, and form layouts.
