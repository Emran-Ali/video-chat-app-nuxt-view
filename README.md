## **Folder Details**

### **1. `output/`**

- **Purpose**: Contains build output or generated files.
- **Usage**: This folder is typically used for storing compiled or processed files, such as those generated by a build tool.

### **2. `assets/`**

- **Purpose**: Stores static assets like CSS, images, and fonts.
- **Subfolders**:
  - **`css/`**: Contains global styles (e.g., `main.css`).
  - **`images/`**: Stores image files used in the application.

### **3. `components/`**

- **Purpose**: Contains reusable Vue components.
- **Subfolders**:
  - **`base/`**: Stores base or generic components (e.g., `ThemeSwitcher.vue`).

### **4. `layouts/`**

- **Purpose**: Contains layout components for the application.
- **Usage**: Layouts define the structure of pages (e.g., header, footer, sidebar).

### **5. `middleware/`**

- **Purpose**: Contains middleware functions for Nuxt.js.
- **Usage**: Middleware can be used for tasks like authentication or logging.

### **6. `node_modules/`**

- **Purpose**: Stores installed npm packages.
- **Usage**: This folder is automatically generated and should not be modified manually.

### **7. `pages/`**

- **Purpose**: Contains application pages.
- **Usage**: Each Vue file in this folder corresponds to a route in the application (e.g., `index.vue` for the home page).

### **8. `pinia/`**

- **Purpose**: Contains Pinia stores for state management.
- **Subfolders**:
  - **`common-stores/`**: Stores shared or common Pinia stores.

### **9. `plugins/`**

- **Purpose**: Contains Nuxt.js plugins.
- **Usage**: Plugins are used to extend Nuxt.js functionality (e.g., `theme.ts` for theme management).

### **10. `public/`**

- **Purpose**: Contains static files that are served directly to the client.
- **Usage**: Files in this folder are accessible at the root URL (e.g., `public/favicon.ico` is accessible at `/favicon.ico`).

### **11. `server/`**

- **Purpose**: Contains server-side code or API routes.
- **Usage**: This folder is used for server middleware or API endpoints.

### **12. `tsconfig.json`**

- **Purpose**: Configuration file for TypeScript.
- **Usage**: Defines compiler options and project settings for TypeScript.

### **13. `themes/`**

- **Purpose**: Contains theme-related files.
- **Files**:
  - **`colors.ts`**: Defines color scales for themes.
  - **`useTheme.ts`**: Composable for managing themes.

### **14. `utils/`**

- **Purpose**: Contains utility functions or helper modules.
- **Usage**: Utilities are reusable functions that can be imported across the application.

### **15. `.gitignore`**

- **Purpose**: Specifies files and directories to be ignored by Git.
- **Usage**: Prevents unnecessary files (e.g., `node_modules/`) from being tracked by Git.

### **16. `.prettierignore`**

- **Purpose**: Specifies files and directories to be ignored by Prettier.
- **Usage**: Prevents Prettier from formatting specific files or directories.
